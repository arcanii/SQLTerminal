# SQLTerminal 0.1.2

## New
- **Automatic updates** via Sparkle — SQLTerminal now checks for and installs
  updates, with a **Check for Updates…** item in the app menu.

## PostgreSQL
- Connection failures now show the server's real reason — authentication
  failures, `pg_hba.conf` rejections, network/SSL errors — instead of a generic
  message. For a "no `pg_hba.conf` entry" rejection it even suggests the exact
  rule to add.
- The connection sheet sizes to the engine so every PostgreSQL field
  (Host/Port/Database/Username/Password) is visible at once, and it remembers the
  engine you last selected.
- `.use` / `.connect` switch databases on your current connection (PostgreSQL
  only), reporting clear "no access" / "does not exist" messages and keeping you
  on the previous database if the switch fails.

## Terminal
- You can now select text in the output pane.
